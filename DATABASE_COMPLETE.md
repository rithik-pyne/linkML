## Known Limitations

### ⚠️ ResponseAssessment Table (1 row only)
**Issue:** Schema has NOT NULL constraints on foreign keys:
```sql
treatment_id TEXT NOT NULL,
imaging_study_id TEXT NOT NULL,
molecular_test_id TEXT NOT NULL,
```

**Impact:** Most response assessments from time-series CSV were filtered out because:
- Some assessments don't have imaging (follow-up visits)
- Some assessments don't have molecular tests (RECIST-only)
- Empty strings in CSV were rejected by NOT NULL constraints

**Solution Options:**
1. Modify schema to make FKs nullable: `treatment_id TEXT` (not NOT NULL)
2. Generate fake molecular test IDs for assessments without ctDNA
3. Use existing molecular_test_id from baseline for all assessments

**Recommended:** Modify schema YAML to make ResponseAssessment FKs optional, then regenerate SQL and re-populate.

## Usage

### Query Examples

**Get all patients:**
```python
import sqlite3
conn = sqlite3.connect('backend/clinical_data.db')
cursor = conn.cursor()
patients = cursor.execute("SELECT * FROM Patient").fetchall()
```

**Get imaging timeline for NGDX-001:**
```python
scans = cursor.execute("""
    SELECT scan_date, imaging_modality, ajcc_stage, primary_tumor_diameter_mm
    FROM ImagingStudy
    WHERE patient_id = 'NGDX-001'
    ORDER BY scan_date
""").fetchall()
```

**Get treatment history:**
```python
treatments = cursor.execute("""
    SELECT treatment_line, drug_name, treatment_start_date, treatment_end_date
    FROM Treatment
    WHERE patient_id = 'NGDX-001'
    ORDER BY treatment_line
""").fetchall()
```

**Get clinical assessment trends:**
```python
labs = cursor.execute("""
    SELECT assessment_date, ecog_status, wbc, hemoglobin, alt
    FROM ClinicalAssessment
    WHERE patient_id = 'NGDX-001'
    ORDER BY assessment_date
""").fetchall()
```
